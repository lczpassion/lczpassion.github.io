我所理解的人生的意义

今天偶然在优酷上看到了宋鸿兵老师的纪录片鸿观的第３１期 : 读反心灵鸡汤 认清自己。这当然不可能是广告，我也不牛逼，宋老师也不需要.注 : 扯的只是个人的观点，如有反对意见可以留言。

目的

我为什么要写这篇博客？其实我早就想写点东西来认识我自己了，人是会不断思考的，思考的东西放在脑子里放多了就需要拿出来整理一下（脑容量小）。而今天看到的鸿观就是一个契机，虽然它开头的几句话有点扯淡，但内容还是不错的（笑）。

自强与好强

开头宋老师讲了两种人：自强与好强的人。就我个人观点来言，还是要去做一个自强的人,每个人不是为别人而活的，而是为自己活的，就像宋老师所说，每个自强的人都有强大的小宇宙，他们不会被外界所干扰,他们是和自己去比而不是和别人去比，与己斗多有意思!

这个社会功利化太重

视频中宋老师说好强的人功利化太重，重不重我不知道，因为我不是一个好强心特别重的人（笑）。但是对于这个功利化，我还是有点想法的。现在国内功利化比较重，好多人想出人头地，这本身没有什么，能混的好为什么不混的好，但是许多人没有底线。视频中宋老师引用了司马光的话”德大于才的叫君子，才大于德的叫小人”。许多人只关心结果而不关心过程，只关心有没有钱，有没有权，对成功的定义就是有多少钱，有多少钱，有钱有权就牛逼，太片面了！５００年以后人们可能不会记住比尔盖茨有多少钱，能被记得的只能是他创建了一个伟大的企业，而金钱只是附属的东西。国内现在创业风这么热，猪能不能飞起来我不知道，我只是知道现在大多是的创业者是为了钱，而不是创建一个伟大的企业，所以它们永远成为不了谷歌，苹果，更不要说超过了。

追寻本心

说了这么多，好像是鸡汤（ｏ（╯□╰）ｏ）。
我所追寻的人生的意义：　做一个自强的人，不趋炎附势，追寻自己的本心，寻找自己真正喜欢的东西，并不断坚持下去。

我们要过的是自己想要的生活，而不是别人眼中好的生活。

Python heapq-堆列算法

This module provides an implementation of the heap queue algorithm, also known as the priority queue algorithm. —来自厚重的Python文档

Heap是一种二叉树，它的每一个父节点的值都小于或等于它的子节点。用数组完成了这个实现，在这个二叉树中，heap[k] <= heap[2*k+1] , heap[k] <= heap[2*k+2] , 即树顶的数是最小的。

heapq　提供了以下几个函数:

  • heapq.heappush(heap,item) : 把 item 的值压入 heap
  • heaoq.heappop(heap) : 从 heao 中 pop 并返回最小的 item , 如果 heap 是空的，IndexError raised,想不用 popping 访问最小的 item , 用 heap[0]
  • heapq.heappushpop(heap,item) : 把 item 压入堆中，然后pop并返回最小的item
  • heapq.heaplfy(x) : 把 list x 转换成　heap
  • heapq.heapreplace(heap,item) : 从 heao 中 pop 并返回最小的 item , 并且压入新的 item
  • heapq.merge(*iterables) : 把多个有序输入转化成一个单一的输出。
  • heapq.nlarget(n,iterable,key=None) : 返回一个有n个最大元素的 list ,如果提供了 key，那么每一个元素就会和 key 进行比较
  • heapq.nsmallest(n,iterable,key=None) : 同 nlarget 相反

The latter two functions perform best for smaller values of n. For larger values, it is more efficient to use the sorted() function. Also, when n==1, it is more efficient to use the built-in min() and max() functions. If repeated usage of these functions is required, consider turning the iterable into an actual heap.

Python Collections模块

官方文档对collections的描述为:This module implements specialized container datatypes providing alternatives to Python’s general purpose built-in containers, dict, list, set, and tuple.

1.deque objects

用法: class collections.deque([iterable][,maxlen]) ,如果maxlen不是具体的或者是None,deque将没有界限，deque不会超过maxlen这个最大值。当deque满了，一旦有新的元素加进来的时候，就会有相应的元素被丢弃。

Deque 对象支持以下方法：

  • append(x) : 把 x 加到 deque 的右侧
  • appendleft(x) : 把 x 加到 deque 的左侧
  • clear() : 删除所有元素
  • count(x) : 计算 deque 元素的数量(这是在3.2后新加进来的)
  • extend(iterable) : 在 deque 的右边加上可迭代的元素
  • extendleft(iterable) : 在 deque 的左边加上可迭代的元素，注意:加上的结果是相反于可迭代的内容
  • pop() : 从 deque 的右边删除并返回一个元素，如果现在没有元素，raise an IndexError
  • popleft() : 从 deque 的左边删除并返回一个元素，如果现在没有元素，raise an IndexError
  • remove(value) : 删除第一次出现的value，如果没有找到，raise ValueError
  • reverse() : 颠倒 deque 中的元素并返回None(这是在3.2后新加进来的)
  • rotate(n) : 从 n 到最右边循环这个 deque，如果 n 是负数，向左循环，向右旋转一步等于d.appendleft(d.pop())
  • Deque　对象还提供了一个只读的属性 : maxlen

你可能会说：没什么特别的啊？

像insert(0,v) , pop(0) 对应着　appendleft() , popleft() ,看起来没什么区别，但实际上还是有区别的。尽管list支持简单的操作，但是它的时间复杂度是O(n),随　n　的增加而增加，而deque则是O(1),所以当项目有时间需求时，还是要用deque的。

2.defaultdict objects

用法: class collections.defaultdict([default_factory[, ...]]) 。第一个参数为default_factory　提供了初始值，它的默认值为None,而dict中不存在默认值说法。除了基础的dict操作外，defaultdict对象还支持以下方法:

missing(key)

如果default_factory属性是None,raise KeyError并把key作为属性。如果不是None,给`key｀提供一个默认的值，插入在字典中并返回它。

4.ChainMap objects

New in version 3.3.
class collections.ChainMap(*maps):支持所有普通的字典方法。

  • maps : 一个用户可以更新的映射表。这个表至少包含一个映射。
  • new_child(m=None) : 返回一个新的ChainMap,如果指定了m,那么m在列表的最前面。
  • parents : 返回一个新的ChainMap包含除了第一个map外的所有maps。
